Features Assign Learn Score Tool Associating File Format

 

Falstaff is an advanced tool for file format identification http://www.forensicswiki.org/wiki/File_Format_Identification.

 

Falstaff takes an innovative scientific approach to format identification. It implements a Maximum Entropy classifier combining features from various sources, such as known format signatures, format extensions but also possibly based on file metadata such as file name, size, dates, Ö into a single evaluation system. In fact any feature that can be programmatically evaluated against a file is a valid input candidate for Falstaffís format identification process. The output of the Falstaff format identification process is a list of possible format matches, with their matching probability score.

 

Falstaff currently identifies more than 1,000 different file formats as specified by the Archive Team file format ontology http://fileformats.archiveteam.org/wiki/File_Formats.

Due to Falstaffís automatic learning capabilities new formats can be added seamlessly. The set of features implemented by the Falstaff classifier includes known format signatures such as Pronom signatures http://apps.nationalarchives.gov.uk/PRONOM[1], but also novel file format signatures extracted automatically from sample files available for the format. Novel signatures are discovered by applying advanced sequence aligning algorithms, which can deal with potential local variations in the file format signature.†

 

In contrast to a traditional ëmagic numberí approach to the file identification problem, where the file is checked for an exact match with a known ëmagicí format signature, Falstaff can reliably identify ëfuzzyí formats such as software source formats. In the case of source files the format features consist of keywords that are used by the different programming languages. There is no guarantee that a particular keyword will be encountered in the source file in a particular programming language, but there is an expectation that some of the keywords used by that language will be found.

 

The Falstaff classifier is trained and tested against sample sets of labelled files.

In the course of the training process each implemented (format, feature) pair is assigned a weight. Features found more frequently in the sample files available for the format are assigned higher weights.

 

The sample files currently used to train Falstaff were collected from public corpora such as Govdoc1 http://digitalcorpora.org/corpora/files/, as well as from a variety of other web sites. Where no real samples are available Falstaff has the ability to simulate file format samples.

 

Falstaff can be recalibrated at any time by retraining and retesting it. Administrative procedures such as classifier training and testing, format sample set and format feature management are also accessible through the Falstaff UI.

 

Falstaff is based on web architecture. There is no need for a local installation nor for transferring the files to be processed to another location. The identification process is carried out locally through the browser (Firefox, Chrome or Explorer).
